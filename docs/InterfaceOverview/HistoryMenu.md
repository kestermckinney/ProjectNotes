# History Menu

## Understanding Page History

The **History** menu provides quick access to recently visited pages. It displays a list of the pages you have opened, allowing you to jump directly to any of them without navigating through menus.

The History menu is separate from the back/forward navigation in the View menu. While the View menu's back/forward arrows let you step through pages one at a time, the History menu shows your complete list of recent pages, so you can jump directly to any page you've visited recently.

## How Page History Works

When you navigate to a page in Project Notes, the application remembers that you opened it. Each entry in the History menu shows:

- **Page title** — The name of the page you visited (for example, "Project Details [P-001 Sample Project]" or "Meeting Notes [2026-03-20]")
- **Timestamp** — The date and time you opened that page (in MM/dd/yy hh:mm format)

The History menu displays the **20 most recent pages** you have visited. If you visit more than 20 different pages, the oldest entry is removed from the list.

**To navigate to a previous page using History:**

1. From the **History** menu, select the page you want to visit.
2. Project Notes opens that page and displays the same record that was open when you last visited it.

## Saving Page State

When you navigate between pages, Project Notes saves the current state of each page to the database. This means that even after closing Project Notes and reopening it, the application remembers:

- Which tab you had selected on each page (for example, the "Tracker" tab on a Project Details page)
- What record was open (for example, which project or meeting notes)
- Your current selections

The next time you open that page, you will see the same data you were viewing before you closed the application.

## Clearing History

The **History** menu is cleared in the following situations:

- **Restarting Project Notes** — When you close and reopen the application, the History menu is empty and starts fresh.
- **Closing a database** — When you close the currently open database, the History menu is cleared.
- **Deleting records** — When you delete a record (such as a project or meeting note), any history entries for that record are automatically removed from the menu.
