# Master Item List Page

## Using the Master Item List Page

The **Master Item List Page** shows every tracker item and action item across all projects in a single view. It is useful for getting a high-level picture of open work, reviewing items assigned to a specific person, or spotting overdue items regardless of which project they belong to.

Because the list spans all projects, some editing capabilities that require a project context are not available here. New items cannot be created from this page; items must be created from within an individual project's [Item Tracker Page](<ItemTrackerPage.md>). Items can still be opened individually in the [Action Item Detail Page](<ActionItemDetailPage.md>) for full editing.

**To navigate to the Master Item List Page:**

1. From the **View** menu, select **Master Item List**.

---

## Master Item List Columns

The table below describes each column. Columns displayed in a shaded font cannot be edited in this view.

| **Column** | **Description** |
| :--- | :--- |
| Project | The project number for the project the item belongs to. This column cannot be edited from the Master Item List. |
| Project Status | The current status of the project (**Active**, **Closed**, etc.). This column cannot be edited. |
| Item | The item number generated by Project Notes. Item numbers start at 0001 and are unique within each project. This column cannot be edited. |
| Type | The item type: **Tracker** (risk, issue, or work item) or **Action** (action item from a meeting). |
| Item Name | A summary title for the item. |
| Identified By | The individual who identified the item. This column cannot be edited in this view. |
| Date Identified | The date the item was identified. |
| Description | A longer description of the item. |
| Assigned To | The individual assigned to resolve the item. This column cannot be edited in this view. |
| Priority | The priority level. Values and their colors are: |

| **Priority** | **Display** |
| :--- | :--- |
| **High** | Displays in red. |
| **Medium** | Displays in yellow. |
| **Low** | Displays in standard font color. |

| **Column** | **Description** |
| :--- | :--- |
| Status | The current status of the item. See [Status Values](#status-values) below. |
| Date Due | The date the item should be resolved by. Overdue dates display in red; dates due today display in yellow. |
| Updated | The date the record was last changed. This column cannot be edited in this view. |
| Date Resolved | The date the item was resolved. Set automatically when **Status** is changed to **Resolved**, but can be edited manually from the [Action Item Detail Page](<ActionItemDetailPage.md>). |
| Comments | A concatenation of all update notes entered on the [Action Item Detail Page](<ActionItemDetailPage.md>). This column cannot be edited. |
| Internal | Indicates whether the item is marked as internal (not visible to clients). Only visible when **Internal Items** is checked in the **View** menu. |

### Status Values

| **Status** | **Description** |
| :--- | :--- |
| **New** | Recently entered; no one has been assigned yet. |
| **Assigned** | Assigned to an individual; work is assumed to be in progress. |
| **Deferred** | Will not be completed during this project. |
| **Resolved** | Completed successfully. Hidden by default unless **Resolved Tracker/Action Items** is checked in the **View** menu. |
| **Canceled** | No longer needed. |

## Differences From the Item Tracker Page

The Master Item List behaves differently from the per-project [Item Tracker Page](<ItemTrackerPage.md>) in several ways:

- **No new items** — New items and copies cannot be created here. Items must be created from within a project.
- **No delete** — Items cannot be deleted from this view.
- **Read-only people columns** — **Identified By** and **Assigned To** show people from all projects, not just the current project's team, and cannot be changed in this view.
- **Read-only project column** — The **Project** column cannot be changed here to prevent accidental reassignment of items between projects.
- **No meeting column** — The **Meeting** column is hidden because meeting context is project-specific.
- **Closed projects included** — Items from closed projects appear in the list (the **Project Status** column makes this visible). The per-project Item Tracker hides closed projects by default.

---

## Filtering and Sorting

The same **View** menu options that apply to the Item Tracker also apply here:

- **Resolved Tracker/Action Items** — Shows or hides resolved, deferred, and cancelled items.
- **Internal Items** — Shows or hides the Internal column and any items marked internal.
- **Closed Projects** — When unchecked, items belonging to closed projects are hidden.

The [Filter Tool](<FilterTool.md>) is available to narrow the list to specific values — for example, to show only items assigned to a particular person or only high-priority items.

Columns can be sorted by clicking their header. See [Sorting and Sizing Columns](<SortingandSizingColumns.md>) for details.

---

## Opening an Item for Full Editing

Double-clicking any row opens the item in the [Action Item Detail Page](<ActionItemDetailPage.md>), where all fields including meeting association, project reassignment, and update notes can be fully edited.
